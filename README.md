# PMSM 제어 · 보드 학습툴

임세영님의 **`PMSM_TI 구현`** 자료(인천대, TMS320F28377D 기반)를 **제어이론도 보드설계도 처음인 사람**이
이해할 수 있도록 만든 통합 학습 프로그램입니다.

---

## 실행 방법

**가장 쉬운 방법 (Windows):** 폴더의 `run.bat` 더블클릭.
환경을 점검한 뒤 GUI가 뜨고, 패키지가 빠졌으면 무엇을 설치하면 되는지 알려줍니다.

**명령줄:**
```bash
cd "C:\Users\user\Desktop\PMSM_학습툴"
python -m pip install -r requirements.txt   # 처음 한 번 (의존성 설치)
python run.py                               # 환경 점검 후 실행
# 또는 점검 없이 바로:  python main.py
```

### 오프라인 PC에서 실행 (인터넷 없는 환경)
인터넷 되는 PC에서 wheel을 미리 받아 함께 복사하면 됩니다.
```bash
# (온라인 PC) 의존성 wheel 다운로드
pip download -r requirements.txt -d wheels
# → 프로젝트 폴더째 USB로 복사 후, (오프라인 PC)에서
python -m pip install --no-index --find-links=wheels -r requirements.txt
python run.py
```

## 프로그램 구성 (탭 3개)

### 1. 제어 시뮬레이터
검증된 PMSM 물리 모델 위에서 **제어 4단계를 직접 돌려봅니다.**
- 왼쪽에서 모드(V/F → I/F → Vector → Speed)와 지령(속도·전류·부하)을 바꾸고
- 가운데 모터 단면에서 **회전자(자석), dq축, 전류 벡터, 전압 벡터**가 도는 걸 보고
- 오른쪽에서 3상 전류 / dq 전류 / 속도 그래프를 실시간으로 확인합니다.

**추천 실험**
1. `Vector` 모드, Iq 지령을 0→2A로 올려보기 → 토크·속도가 커진다 ("Iq가 토크다").
2. `Speed` 모드, 속도 300rpm, 부하 토크를 0→50mN·m → iq가 자동으로 늘어 속도를 지킨다.
3. `V/F` 모드 → 제어 없이 그냥 도는, 가장 원시적인 방식 체험.

### 2. 보드 · 소자 구조
실제 FOC 보드(Cheap FOCer 2 / TI 28377D)의 **블록도를 클릭**하면 각 소자가
**왜 필요하고 / 무슨 기능을 하고 / 신호가 어떻게 흐르는지** 설명이 뜹니다.
선 색: 🔴전력(큰 전류) · 🔵제어(PWM) · 🟢센싱 · 🟠통신.

### 3. 통신 방식
**통신이 왜 필요한지**부터, UART/SPI/I2C/CAN 비교표와 각 방식 설명,
그리고 **이 모터 보드에서 무엇을 써야 하는지**(엔코더=SPI, 상위제어=CAN) 정리.

## 핵심 개념 한 장 요약 (제어이론 처음이라면 이것만)

**모터를 잘 돌린다 = 전류를 원하는 대로 흘린다.** FOC(벡터제어)는 이렇게 합니다.

1. **측정**: 상전류(ia,ib,ic)를 션트저항+ADC로 읽고, 회전자 각도(θe)를 엔코더로 읽는다.
2. **변환(Clarke→Park)**: 3상 전류를 회전자와 함께 도는 dq 좌표로 바꾼다.
   - `id` = 자석 방향 전류 (보통 0으로 둠, 토크에 기여 안 함)
   - `iq` = 자석과 직각 전류 = **토크를 만드는 전류**
3. **제어(PI)**: `id→0`, `iq→원하는 토크전류`가 되도록 전압(Vd,Vq)을 계산.
4. **역변환(역Park→SVPWM)**: 전압을 다시 3상으로 바꿔 **ePWM 듀티**로 출력.
5. 이 1~4를 **인터럽트로 1초에 수만 번**(예 20kHz) 반복한다.

> dq로 바꾸는 이유: 도는 3상 교류(sin)를 그대로 제어하면 어렵지만, 회전자와 같이 도는
> dq 좌표에서 보면 정상상태에서 **값이 가만히 있는 직류**처럼 보여 PI 제어가 쉬워집니다.

### 제어 4단계 (PDF 5p와 동일)
| 단계 | 이름 | 한 줄 | 각도 출처 |
|---|---|---|---|
| 1 | V/F | 주파수에 비례해 전압↑, 억지로 구동 | 지령(개루프) |
| 2 | I/F | 전류 크기 제어 + 각도는 천천히 회전 | 지령(개루프) |
| 3 | Vector(FOC) | 전류를 dq로 분리해 정밀 제어 | **실제 회전자각** |
| 4 | Speed | 바깥 속도루프가 Iq 지령 생성(이중 루프) | 실제 회전자각 |

## 소자 빠른 정리 (왜 필요한가)

| 소자 | 왜 필요 |
|---|---|
| MCU (28377D) | 모든 측정·계산·PWM 출력. 모터제어 특화 DSP |
| 게이트 드라이버 (DRV8301) | MCU의 약한 신호로 큰 MOSFET을 빠르게 ON/OFF |
| MOSFET ×6 | DC를 잘라 3상 교류 합성 (데드타임 필수) |
| 션트저항+OP-AMP | 전류를 전압으로 바꿔 측정 (작은 저항=손실↓, 그래서 증폭 필요) |
| 엔코더 (AS5047P) | 회전자 각도 θe 측정 — Park 변환의 필수 입력 |
| 벌크 커패시터 | 스위칭 전류 요동 흡수, DC 전압 안정화 |
| CAN 트랜시버 (TJA1051) | 상위 제어기와 명령·상태 통신(노이즈 강함) |

## TI 자료(PDF)와의 연결 지도
- **PWM 듀티/데드타임** → 보드탭 '인버터', PDF 24·31p (ePWM submodule)
- **ADC 오프셋·스케일·Calibration** → 보드탭 '전류센서', PDF 36~38p
- **인터럽트(PIE/IER/Fsamp)** → 보드탭 'MCU', PDF 44~52p
- **각도 추정/보상** → 보드탭 '엔코더', PDF 8p·78p
- **통신 프로토콜/플랫폼** → 통신탭, PDF 76p
